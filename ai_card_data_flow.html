카드 앱 파일을 보관하는 곳
활동기록카드의 HTML·CSS·JS 소스 코드가 저장됩니다. 어드민이 코드를 수정하면 GitHub에 커밋하고, 변경사항이 자동으로 반영됩니다.
팀원이 카드를 여는 주소를 제공
GitHub Pages가 index.html을 인터넷에 공개합니다. 팀원은 별도 설치 없이 브라우저에서 카드를 바로 작성할 수 있습니다.
데이터는 저장하지 않음
GitHub는 앱 파일(코드)만 담습니다. 실제 카드 내용(데이터)은 저장하지 않으며, 데이터 저장은 Supabase가 담당합니다.
비유하자면
GitHub는 건물(앱 구조)을 짓고 주소를 부여하는 역할.
실제 데이터(카드 내용)는 건물 안의 금고(Supabase)에 보관.
Supabase
데이터베이스 · 인증 · 서버 함수
카드 데이터를 저장하는 금고
작성자가 저장 버튼을 누르면 카드의 모든 내용(단계·도구·WF·인사이트 등)이 JSON 형태로 Supabase 데이터베이스에 저장됩니다.
공유 링크로 데이터를 꺼내주는 역할
열람자가 공유 링크를 열면, URL의 고유 ID로 Supabase에 요청해 해당 카드 데이터를 불러옵니다. 이 데이터로 뷰어 모드가 구성됩니다.
AI 자동 채우기 중간 다리 역할
Edge Function(gemini-proxy)이 브라우저 대신 Groq AI API를 호출합니다. 보안 키를 서버에서 안전하게 관리하고 CORS 문제를 해결합니다.
비유하자면
Supabase는 우체국+금고+직원 역할.
데이터를 보관(금고)하고, 링크로 꺼내주고(우체국), AI 요청도 대신 처리(직원).
